## Principal finding

KL2003 Theorem 3.1 constructs a finite final inequality `I_k^m(EL)` and claims
that its tree is independent of splitting order. The implemented route proves
a different, weaker syntactic statement and a sufficient semantic statement:
for each fixed source system, mode, and `y >= 2`, it constructs an actual
finite scheduler run plus a critical assignment whose selected expression is
all-retarded and lies below the source root value.

This is not a formalization of canonical-normal-form uniqueness. It is a
proof re-architecture based on the paper's critical-assignment semantics in
equations (304)-(305). The distinction is already stated in the module header
and is now also recorded in the original general-k scoping gate.

## Why the pointwise route is sufficient

The downstream chain does not silently treat a pointwise witness as a fixed
EL tree:

1. `SourceELRetardedWitness` retains the scheduler step, provenanced generated
   tree, critical assignment, criticality proof, negative-shift proof, and
   source-value inequality.
2. `sourceELRetardedWitness_coefficient_feasible` proves coefficient
   feasibility for that exact witness. Splitting and deletion are monotone for
   certificate evaluation, and selecting one branch from each minimum can only
   increase the coefficient expression.
3. `exists_uniform_criticalSelectedDepthBound` separately rules out witnesses
   of unbounded selected depth over all modes and all `y >= 2` by a finite
   inverse-limit/Konig argument and the recurrent advanced-arrival descent.
4. `exists_uniform_sourceELRetardedWindow` converts that depth bound into one
   positive retarded window valid for every pointwise witness.
5. The dynamic retarded induction chooses the exact witness for the current
   mode and argument; it never requires order independence or one global
   materialized tree.

## Adversarial checks

1. A witness cannot be supplied as an arbitrary retarded expression; it must
   be generated by `GeneralKCriticalScheduler.run` from the initial source
   tree.
2. `assignment.selectedExpr_eval_eq` preserves the source value because the
   assignment is critical for `sourcePhiK` at the current `y`.
3. Every selected leaf is proved strictly negative, not merely
   non-expandable under an unrelated predicate.
4. Coefficient feasibility does not assume that the source-critical branch is
   coefficient-critical. It uses the correct one-way inequality from the full
   minimum expression to the selected coefficient expression.
5. The row inequality remains in the correct direction:
   `selectedExpr.eval sourcePhiK y <= sourcePhiK root.mode (...)`.
6. Initial arguments are nonnegative and source rows are used only for
   `y >= 2`; the negative zero-extension of `sourcePhiK` is not used to justify
   a source expansion.
7. Uniformity is not inferred from pointwise termination. It is proved later
   in dedicated `UniformCriticalDepth` and `UniformRetardedWindow` modules.
